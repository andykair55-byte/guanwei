# PC 端前端优化计划

## 问题诊断

当前 PC 端存在以下问题：
1. **左右侧边栏字体过小** — `DesktopSidebar` 使用 `text-[13px]`/`text-[12px]`/`text-[11px]`，`DesktopRightPanel` 类似
2. **社区瀑布流只有 2 列** — `CommunityPage.tsx` 手动分为 `leftCol`/`rightCol` 两列，PC 宽屏浪费空间
3. **子页面全部按移动端设计** — 辩论场、求证工具子页面都用 `max-w-[480px] mx-auto` 限宽，PC 端像手机模拟器
   - `DebateLobby.tsx` — `max-w-[480px]`
   - `DebateRoomPage.tsx` — `max-w-[480px]`
   - `DebatesPage.tsx` — 无 max-width 但单列
   - `AIBattle.tsx` — 无 max-width 但单列
   - `ExifAnalyzer.tsx` — 无 max-width，单列
   - `EmotionDetector.tsx` — 无 max-width，单列
   - `ReverseImageSearch.tsx` — 无 max-width，单列
   - `TimelineBuilder.tsx` — 无 max-width，单列
   - `PlagiarismChecker.tsx` — 无 max-width，单列
   - `MultiSourceVerify.tsx` — 无 max-width，单列
   - `RoundTable.tsx`

---

## 改动计划

### 1. 侧边栏字体放大 + 间距优化

**文件**: `src/components/DesktopSidebar.tsx`
- 主导航文字: `text-[13px]` → `text-sm` (14px)
- 工具箱标题: `text-[11px]` → `text-xs` (12px)
- 工具箱项: `text-[12px]` → `text-sm` (14px)
- Logo 标题: `text-[18px]` → `text-lg` (18px，保持)
- Logo 副标题: `text-[11px]` → `text-xs` (12px)
- 底部徽章文字: `text-[11px]` → `text-xs` (12px)
- 侧边栏宽度: `w-[200px]` → `w-[220px]`

**文件**: `src/components/DesktopRightPanel.tsx`
- 标题: `text-[14px]` → `text-sm` (14px，保持)
- 辩论话题: `text-[12px]` → `text-sm` (14px)
- 热搜话题: `text-[12px]` → `text-sm` (14px)
- 排名数字: `text-[11px]` → `text-xs` (12px)
- 数据标签: `text-[10px]` → `text-[11px]`

### 2. 社区瀑布流改为 3 列

**文件**: `src/pages/CommunityPage.tsx`
- 移除手动 `leftCol`/`rightCol` 两列分组逻辑
- PC 端改用 CSS Grid: `grid grid-cols-2 lg:grid-cols-3 gap-4`
- 移动端保持当前两列瀑布流不变
- 新增 `useIsDesktop()` hook（复用 MelonFieldPage 的模式）
- 桌面端移除 `sticky top-0` 的 glass header 中的移动端限制样式

**文件**: `src/components/CommunityCard.tsx`
- 无结构变化，Grid 布局自动适配

### 3. 辩论场子页面 PC 端适配

**文件**: `src/pages/DebateLobby.tsx`
- PC 端: 移除 `max-w-[480px]` 限制，内容自然撑满 Layout 中间栏
- PC 端: 头部 `flex items-center` 放大字号，去掉移动端紧凑间距
- PC 端: 房间卡片改为 Grid 2 列排布 `grid grid-cols-1 lg:grid-cols-2 gap-4`
- 移动端: 保持原样

**文件**: `src/pages/DebateRoomPage.tsx`
- PC 端: 移除 `max-w-[480px] mx-auto`，使用两栏布局
  - 左栏: 舞台区 + 发言流（占主区域）
  - 右侧信息面板: 观众列表、阶段信息、操作按钮
- 移动端: 保持原样

### 4. 工具子页面 PC 端适配

这些页面都有返回按钮 + 标题头部，PC 端需要更合理的宽度利用。

**通用策略**: 对所有工具页面统一添加 `useIsDesktop()` + 内容区 `max-w` 条件判断。

**文件**:
- `src/pages/ExifAnalyzer.tsx` — PC 端内容区 `max-w-3xl mx-auto` (768px)，合理居中
- `src/pages/EmotionDetector.tsx` — PC 端内容区 `max-w-3xl mx-auto`
- `src/pages/ReverseImageSearch.tsx` — PC 端内容区 `max-w-3xl mx-auto`
- `src/pages/TimelineBuilder.tsx` — PC 端内容区 `max-w-3xl mx-auto`
- `src/pages/PlagiarismChecker.tsx` — PC 端内容区 `max-w-3xl mx-auto`
- `src/pages/MultiSourceVerify.tsx` — PC 端内容区 `max-w-3xl mx-auto`

**文件**: `src/pages/DebatesPage.tsx` — PC 端内容区 `max-w-3xl mx-auto`

**文件**: `src/pages/AIBattle.tsx` — PC 端内容区 `max-w-4xl mx-auto` (896px，辩论内容需要更多空间)

**文件**: `src/pages/RoundTable.tsx` — PC 端内容区 `max-w-3xl mx-auto`

**文件**: `src/pages/DebateRoomPage.tsx` — PC 端 `max-w-4xl mx-auto`

### 5. 头部标题在 PC 端放大

所有子页面的 `<h1>` 标题在 PC 端适当放大:
- 移动端 `text-[16px]` → PC 端 `text-xl` 或 `text-lg`
- 移动端 `text-[18px]` → PC 端 `text-2xl`

## 验证方式

1. `npm run lint` — 无新增 error
2. 浏览器窗口宽度 ≥ 768px:
   - 侧边栏文字清晰可读
   - 社区页显示 3 列瀑布流
   - 辩论场/工具子页面内容合理居中，不显得拥挤
3. 浏览器窗口宽度 < 768px (或真机模拟):
   - 所有页面保持原有移动端布局不变
