## 项目总览

**一句话描述**：我们让AI陪用户分析热点，提升全民信息素养。

**用户为什么今天会打开它**（第一触发场景）：

好玩。抖音求真、腾讯较真缺少用户参与环节，本质是高级豆包，问完即走。观微做了大量的任务设计和娱乐设计。

首先考察的是互联网用户爱吃瓜的特性。页面有时间线可视化收集信息，有社区分享互助，有瓜田。本质是以一种娱乐轻松的方式学习。设计了工作间，任何想写帖子的用户门槛和效率都得到提升——大部分人不是不会写，而是没时间整理收集、想文案。这一点知乎、小红书、微博都没有做到，本质还是依靠大量媒体工作者。大量媒体内容良莠不齐，观微利用LLM天然的信息整理能力完成，把Agent的能力充分融入项目，并不违和。

**用户完成什么事情后会觉得今天没白来**（核心任务）：

项目里设计了段位系统和辩论房系统以及求助帖。段位系统可以升级产生正反馈。辩论房可以人机对决、人人对决、看AI和AI对决辩论。这是一个可能会爆的点——谁不想看孔子和诸葛亮辩论呢？当前互联网环境需要的是流量和热度，利用LLM的随机性如果能爆出一场高质量的辩论赛即可。抖音里辩论的流量可不低，几亿。就每天直播间挂一份标题"计算机还有必要学吗"，都能找到类似的直播间，几百人看，几个人辩论。辩论视频的流量更不低了。

热点1是这个。热点2像梗指南、科普营销号，本质没有什么技术门槛却有几十万播放，粉丝几百万。观微要嵌入竞品分析功能，利用LLM天然的信息搜索和处理能力。这个渠道未来肯定会被AI代替的——他们不过是用视频将信息整理后以趣味的方式解释，有用户留存。抖音求真、腾讯较真无用户互动。知乎、微博用户体系明显，一个高知一个女性向。知乎靠的是给创作者盈利知识付费留人。现在LLM足以打破大量信息差，这是其他平台没有的，观微先做出来了。

**目标用户**：大学生，对信息最渴望、最有时间、愿意搭建社区的群体。校园瓜和社会瓜往往是不需要答案的，这类群体会持续追踪，且有能力利用LLM写出高质量帖子。

**为什么不用微博、知乎、小红书，而要打开观微**：定位不同，观微走的是自己的路。

**如果只能保留一个模块**：所有的设计是相辅相成的，逻辑闭环。

**技术栈**：

| 层 | 技术 |
|---|------|
| 前端 | React 19 + TypeScript 6 + Vite 8 + Tailwind v4 + Zustand 5 + react-router-dom 7 |
| 后端 | FastAPI (Python 3.12+) + SQLAlchemy + PostgreSQL + Redis |
| AI Pipeline | LangGraph DAG + Commander 容错内核（AgentPool + Checkpoint + 热替换） |
| AI 引擎 | Mirror — 核心基座能力架构，Commander/AgentPool/Checkpoint 模式抽取到 backend |
| 部署 | Docker Compose（PostgreSQL + Redis） |

**规模**：前端约 25,000 行源码（43 个页面 + 组件/服务/Store），后端约 5,000 行 Python

## 一、产品哲学（读完即理解观微在做什么）

这不是一套锦上添花的理念，而是做每一个模块时的判断依据。不理解这五点，就看不懂任何一个功能为什么存在。

**热点是流量入口。** 人不是为了 AI 来，而是为了热点、为了吃瓜来的。

**娱乐是留存手段。** 猜真假、段位、辩论房、AI 对战，这些不是附属功能，而是用户停留的原因。

**AI 是能力，不是主角。** AI 的职责是整理信息、降低创作门槛，而不是取代用户。

**社区是价值沉淀。** 用户不是问完即走，而是留下高质量内容，让后来的用户继续消费、继续讨论。

**信息素养是最终目标。** 但它不是通过上课获得，而是在娱乐、参与、创作的过程中自然形成。

## 二、产品飞轮：每个模块的设计逻辑

### 1. 瓜田不是"猜对错"，而是"让更好的信息浮上来"

很多热点信息没有所谓的对错。每个人视角不同、信息量不同，得到的结论也不同。如果瓜田的核心是"猜对→奖励，猜错→惩罚"，那就把信息素养降级成了答题器——用户只需要猜中"正确答案"就行，根本不需要思考。

瓜田的真正设计是：

**佐证就是见微版的 Community Note。** 用户针对一个瓜贴出高质量信息，不是批改别人的答案，而是给所有人多一个参考角度。用户看完佐证的反应不应该是"哦我错了"，而应该是"原来还有这个角度 / 这个信息我之前不知道"。

**高质量佐证怎么来的？通过工作间。** 大部分人不是不会写，而是没时间整理收集、想文案。工作间用AI把"写一篇信息密度高的内容"的门槛从"需要花2小时查资料"降到"给个主题就能生成骨架"，让更多用户能参与内容生产。

**低质量佐证怎么办？点赞/踩体系，良币驱逐劣币。** 本质和评论点赞的排序是一样的——好的信息被顶上去，差的信息沉下去。不需要人工审核，不需要编辑判断，社区自己完成筛选。

**这就形成了一个完整的飞轮：**

```
吃瓜天性 → 进入瓜田 → 看到佐证(Community Note) → 发现"原来还有这个角度"
  → 想参与讨论 → 工作间降低写帖门槛 → 产出高质量佐证 → 被点赞 → 获得正反馈 → 继续参与
```

每个环节都指向信息素养的提升，但不是通过"上课"或"被纠正"实现的——而是在娱乐、参与、创作的过程中自然形成的。

### 2. 娱乐模块与瓜田的分工不同

原来"猜错扣积分，弹出科普视频"的设计是一个偏差。这个设计更适合娱乐模块：

**评论判官模式（类似美团评论判官）：** 给一个争议场景（比如"商家和用户谁奇葩"），让用户投票判断。这是一个容易复现且真正合理的点——结果即时可见，用户看完就有反馈闭环。而且内容供给成本低，真实评论争议每天都有。

**瓜田信息流不适合做这个。** 瓜田的目的是提升用户信息素养，不要被风向带着走。娱乐模块的判官模式是"娱乐向的惩罚机制"，跟瓜田的定位不同。

### 3. 辩论房——社交裂变引擎

辩论房（人机对决、人人对决、AI vs AI）是潜在的流量爆发点。谁不想看孔子和诸葛亮辩论呢？

抖音里辩论的流量以亿计——"计算机还有必要学吗"这类标题的直播间，几百人看、几个人辩论，辩论视频的流量更不低。利用LLM的随机性，如果能爆出一场高质量的辩论赛，就是天然的传播素材。

段位系统提供留存动力：7段位晋升正反馈，驱动用户持续参与。"前期快后期慢"——新手1-2周就能升级，顶级按年计，稀缺感=价值感。

### 4. 梗指南/科普营销号——AI可以替代的赛道

梗指南、科普营销号本质没有什么技术门槛，却有几十万播放、几百万粉丝。他们不过是用视频将信息整理后以趣味的方式解释。

观微嵌入竞品分析功能，利用LLM天然的信息搜索和处理能力，直接在这个渠道上做出差异化——这个渠道未来肯定会被AI代替的，观微先做出来了。

### 5. 完整的飞轮总结

```
热点(流量入口) → 瓜田(消费) → 佐证/工作间(生产) → 社区(沉淀)
                                ↓
                          辩论房(裂变)
                                ↓
                          段位(留存)
```

- 热点是流量入口：人不是为了AI来，而是为了热点、为了吃瓜来的
- 娱乐是留存手段：猜真假、段位、辩论房、AI对战不是附属功能，而是用户停留的原因
- AI是能力不是主角：AI的职责是整理信息、降低创作门槛，而不是取代用户
- 社区是价值沉淀：用户不是问完即走，而是留下高质量内容，让后来的用户继续消费、继续讨论
- 信息素养是最终目标：但不是通过上课获得，而是在娱乐、参与、创作的过程中自然形成

### 🔴 关键隐藏复杂度

#### 1. LLM 额度消耗不可控
工作间骨架生成 + trae宝对话 + 娱乐大厅 AI 辩论，三个模块同时在线时对 LLM API 额度消耗非常大。降级策略有设计（预设模板），但"骨架生成换成模板"的体验落差可想而知。

#### 2. 多人辩论被低估
虽然 MVP 标记"即将开放"占位，但一旦开放，WebSocket 实时辩论涉及：
- 多客户端状态同步（6 个席位 + 发言顺序 + 计时器）
- 掉线重连后状态恢复
- 并发发言冲突处理
- 抬走投票的实时计票

复杂度远高于当前预期。

#### 3. 实时状态的持久化盲区
- zustand 的 persist 只保存了 LLM 配置和草稿
- 辩论/下注等实时状态在页面关闭或网络断开后丢失
- 没有设计任何恢复机制
