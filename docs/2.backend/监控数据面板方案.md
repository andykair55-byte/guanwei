# 监控数据面板方案

## 1. 需求概述

根据项目需求，需要设计并开发后端可视化数据面板，用于监控系统运行状态、用户行为数据及业务指标。

### 监控目标

| 维度 | 目标 |
|------|------|
| **系统监控** | 服务可用性、响应时间、错误率 |
| **用户行为** | 日活、留存率、转化率 |
| **业务指标** | 猜瓜参与率、正确率、积分消耗 |
| **AI 性能** | LLM 调用次数、成本、响应时间 |

---

## 2. 监控指标定义

### 2.1 系统运行状态

| 指标 | 定义 | 单位 | 告警阈值 |
|------|------|------|---------|
| **服务可用性** | 成功请求 / 总请求 | % | < 99% |
| **平均响应时间** | 所有请求响应时间均值 | ms | > 500ms |
| **P95 响应时间** | 95% 请求响应时间 | ms | > 1000ms |
| **错误率** | 错误请求 / 总请求 | % | > 1% |
| **CPU 使用率** | CPU 占用率 | % | > 80% |
| **内存使用率** | 内存占用率 | % | > 85% |
| **磁盘使用率** | 磁盘占用率 | % | > 90% |
| **数据库连接数** | 当前连接数 / 最大连接数 | % | > 80% |
| **Redis 内存** | Redis 内存占用 | MB | > 90% |

### 2.2 用户行为数据

| 指标 | 定义 | 单位 | 计算方式 |
|------|------|------|---------|
| **日活用户** | 当日有活跃行为的用户数 | 人 | 去重 |
| **周活用户** | 7日内有活跃行为的用户数 | 人 | 去重 |
| **月活用户** | 30日内有活跃行为的用户数 | 人 | 去重 |
| **新增用户** | 当日注册的用户数 | 人 | 注册时间筛选 |
| **留存率 D1** | 次日留存用户 / 当日新增用户 | % | 次日回访 |
| **留存率 D7** | 7日留存用户 / 当日新增用户 | % | 7日回访 |
| **留存率 D30** | 30日留存用户 / 当日新增用户 | % | 30日回访 |
| **平均停留时长** | 每次会话平均时长 | 分钟 | 总时长 / 会话数 |
| **人均浏览量** | 人均浏览页面数 | 页 | 总浏览量 / 用户数 |

### 2.3 业务指标

| 指标 | 定义 | 单位 | 计算方式 |
|------|------|------|---------|
| **猜瓜参与率** | 参与猜瓜用户 / 总活跃用户 | % | 参与数 / 活跃数 |
| **平均正确率** | 正确猜瓜次数 / 总猜瓜次数 | % | 正确数 / 总数 |
| **段位分布** | 各段位用户占比 | % | 各段位人数 / 总人数 |
| **积分获取** | 当日总获取积分 | 分 | 积分记录求和 |
| **积分消耗** | 当日总消耗积分 | 分 | 积分记录求和 |
| **积分余额** | 用户平均积分余额 | 分 | 总余额 / 用户数 |
| **佐证提交率** | 提交佐证用户 / 参与猜瓜用户 | % | 提交数 / 参与数 |
| **佐证点赞率** | 点赞佐证数 / 总佐证数 | % | 点赞数 / 总佐证数 |
| **最佳佐证数** | 当日被选为最佳的佐证数 | 个 | 计数 |

### 2.4 AI 性能指标

| 指标 | 定义 | 单位 | 计算方式 |
|------|------|------|---------|
| **求证调用次数** | 当日 AI 求证调用次数 | 次 | 计数 |
| **平均响应时间** | AI 求证平均响应时间 | ms | 总耗时 / 调用数 |
| **成功率** | 成功调用 / 总调用 | % | 成功数 / 总数 |
| **模型分布** | 各模型调用占比 | % | 各模型调用数 / 总数 |
| **Token 消耗** | 当日总 Token 消耗 | 千 | Token 计数 |
| **预估成本** | 当日 AI 调用预估成本 | 元 | Token × 单价 |

## 3. 数据采集方案

### 3.1 采集方式

| 数据类型 | 采集方式 | 工具 |
|----------|---------|------|
| **系统指标** | 定时采集 | Prometheus / 自定义脚本 |
| **用户行为** | 埋点上报 | 前端 SDK + 后端日志 |
| **业务指标** | 数据库查询 | 定时任务 + SQL |
| **AI 指标** | API 日志 | LLM 服务层埋点 |

### 3.2 数据存储

| 数据类型 | 存储方案 | 保存周期 |
|----------|---------|---------|
| **实时指标** | Redis | 1小时 |
| **分钟级指标** | InfluxDB | 7天 |
| **小时级指标** | PostgreSQL | 30天 |
| **日级指标** | PostgreSQL | 永久 |
| **原始日志** | Elasticsearch | 30天 |

### 3.3 采集频率

| 指标类型 | 频率 | 说明 |
|----------|------|------|
| **系统状态** | 1分钟 | CPU/内存/磁盘 |
| **API 响应** | 实时 | 请求日志 |
| **用户行为** | 实时 | 埋点上报 |
| **业务指标** | 1小时 | 汇总统计 |
| **AI 性能** | 实时 | LLM 调用日志 |
| **日级汇总** | 每日凌晨 | 日活/留存/积分 |

## 5. 后端实现方案

### 5.1 数据模型

```python
class Metric(Base):
    """监控指标表"""
    __tablename__ = "metrics"
    
    id = Column(Integer, primary_key=True)
    name = Column(String(50))          # 指标名称
    category = Column(String(20))      # 类别：system/user/business/ai
    value = Column(Float)              # 指标值
    unit = Column(String(10))          # 单位
    timestamp = Column(DateTime)       # 采集时间
    dimensions = Column(JSON)          # 维度（如模型名称、段位等）

class DailyStat(Base):
    """日统计数据表"""
    __tablename__ = "daily_stats"
    
    id = Column(Integer, primary_key=True)
    date = Column(Date, unique=True)
    active_users = Column(Integer)
    new_users = Column(Integer)
    total_users = Column(Integer)
    guess_count = Column(Integer)
    correct_count = Column(Integer)
    point_earned = Column(Integer)
    point_spent = Column(Integer)
    verification_count = Column(Integer)
    evidence_count = Column(Integer)
    created_at = Column(DateTime, default=datetime.utcnow)
```

## 7. 告警机制

### 7.1 告警规则

| 指标 | 阈值 | 告警级别 | 通知方式 |
|------|------|---------|---------|
| 服务可用性 < 99% | 持续 5 分钟 | 严重 | 钉钉 + 邮件 |
| 错误率 > 1% | 持续 5 分钟 | 警告 | 钉钉 |
| CPU > 80% | 持续 10 分钟 | 警告 | 钉钉 |
| 内存 > 85% | 持续 10 分钟 | 警告 | 钉钉 |
| 响应时间 > 500ms | 持续 5 分钟 | 警告 | 钉钉 |
| LLM 成功率 < 95% | 持续 5 分钟 | 严重 | 钉钉 + 邮件 |

### 7.2 告警通知

| 方式 | 适用场景 | 配置 |
|------|---------|------|
| **钉钉群消息** | 即时告警 | Webhook 机器人 |
| **邮件** | 严重告警 | SMTP 配置 |
| **短信** | 紧急告警 | 短信 API |

### 7.3 告警收敛

| 规则 | 说明 |
|------|------|
| **重复告警** | 同一告警 5 分钟内只通知一次 |
| **告警升级** | 持续 15 分钟未恢复，升级为严重告警 |
| **告警抑制** | 维护期间自动抑制告警 |

## 8. 部署方案

### 8.1 独立部署

监控面板作为独立应用部署，与主应用分离。

```
┌─────────────────────────────────────────────────────┐
│                   Nginx                            │
│  /dashboard → Dashboard 应用                       │
│  /api → 主应用 API                                 │
└─────────────────────────────────────────────────────┘
```

### 8.2 访问控制

| 方式 | 说明 |
|------|------|
| **IP 白名单** | 仅允许公司 IP 访问 |
| **Basic Auth** | 用户名密码认证 |
| **JWT 认证** | 与主应用共享认证 |

### 8.3 性能优化

| 优化项 | 方案 |
|--------|------|
| 数据缓存 | Redis 缓存查询结果 |
| 图表懒加载 | 按需加载图表组件 |
| 静态资源 CDN | 前端静态资源加速 |

## 10. 风险评估

### 10.1 技术风险

| 风险 | 等级 | 缓解措施 |
