## 一、模块定位

**求证工具集是观微的AI辅助验证工作台——用户输入可疑信息，通过EXIF分析、情绪检测、LLM文字求证三个工具获得验证结论，结果可一键提交为瓜田佐证。**

| 聚合报告+可信度评分 | 依赖被砍掉的工具，无法聚合 |
| 工具使用统计 | MVP无必要 |

### 3.3 CORS问题解决方案

**问题：** 用户输入外部URL后，前端直接fetch会因CORS被浏览器拦截。

**方案：后端代理。**

```python
# backend/app/utils/cors_proxy.py
from fastapi import APIRouter, HTTPException
import httpx

router = APIRouter()

@router.post("/api/tools/fetch-url")
async def fetch_url(url: str):
    """
    后端代理抓取外部URL内容，解决CORS问题。

    限制：
    - 只允许http/https协议
    - 响应体最大1MB
    - 超时10秒
    - 每用户每分钟最多10次请求
    """
    if not url.startswith(("http://", "https://")):
        raise HTTPException(400, "只支持http/https链接")

    async with httpx.AsyncClient(timeout=10, follow_redirects=True) as client:
        try:
            resp = await client.get(url, headers={"User-Agent": "JianWei/1.0"})
            content = resp.text[:1_000_000]  # 限制1MB
            return {"content": content, "status": resp.status_code}
        except httpx.TimeoutException:
            raise HTTPException(504, "抓取超时")
        except Exception as e:
            raise HTTPException(500, f"抓取失败: {str(e)}")
```

### 3.4 LLM文字求证的完整实现路径

#### 3.4.1 输入

用户输入：文本（直接输入）或 URL（后端代理抓取后提取正文）

#### 3.4.2 输入预处理

```python
def preprocess_input(user_input: str, input_type: str) -> str:
    """
    预处理用户输入，构造LLM的输入文本。

    - 直接文本：截断到2000字（约3000 token，留足prompt空间）
    - URL内容：用httpx抓取后，提取HTML正文（readability算法或简单去标签），截断到2000字
    """
    if input_type == "url":
        # 调用cors_proxy抓取
        raw_html = fetch_url_content(user_input)
        # 简单提取正文：去除script/style标签，取body文本
        text = extract_text_from_html(raw_html)
    else:
        text = user_input

    # 截断
    return text[:2000]
```

#### 3.4.3 Prompt构造

```python
VERIFY_PROMPT = """你是一个信息求证助手。请分析以下信息的可信度。

## 分析要求
1. 判断该信息是"基本属实"、"部分属实"、"难以判断"还是"基本不实"
2. 列出支持该信息的关键论据（最多3条）
3. 列出质疑该信息的关键疑点（最多3条）
4. 给出1-5星的可信度评级

## 输出格式（严格JSON）
{
  "verdict": "基本属实|部分属实|难以判断|基本不实",
  "stars": 1-5,
  "supporting": ["论据1", "论据2", "论据3"],
  "questioning": ["疑点1", "疑点2", "疑点3"],
  "summary": "50字以内的总结"
}

## 待分析信息
{text}"""
```

#### 3.4.4 Groq调用

```python
from groq import Groq

client = Groq(api_key=settings.GROQ_API_KEY)

def call_llm_verify(text: str) -> dict:
    """
    调用Groq进行文字求证。

    Token预算：
    - 输入：~3000 token（2000字中文 + prompt模板）
    - 输出：~500 token
    - 单次总计：~3500 token
    - 每用户每日限10次 = 35000 token/日，远在免费额度内
    """
    response = client.chat.completions.create(
        model="llama-3.3-70b-versatile",
        messages=[{"role": "user", "content": VERIFY_PROMPT.format(text=text)}],
        temperature=0.3,  # 低温度，结果更稳定
        max_tokens=500,
        response_format={"type": "json_object"},  # 强制JSON输出
    )
    return json.loads(response.choices[0].message.content)
```

```
IDLE（空闲）→ FETCHING（抓取URL中）→ ANALYZING（LLM分析中）→ DONE（完成）

```

|------|--------|

## 六、异常场景应对

### 6.1 LLM超时

| 场景 | 处理 |
|------|------|
| Groq API >15秒无响应 | 后端断开连接，返回504，前端提示"分析超时，请稍后重试" |
| 连续3次超时 | 前端提示"AI服务暂时不可用，请使用其他工具" |

### 6.2 LLM返回格式错误

| 场景 | 处理 |
|------|------|
| 返回非JSON | 后端catch解析异常，返回500，前端提示"分析结果解析失败，请重试" |
| JSON缺少必要字段 | 后端校验字段完整性，缺失则用默认值填充（verdict="难以判断", stars=3） |
| response_format不生效 | 备选方案：在prompt中强调JSON格式，并在后端做正则提取 |

### 6.3 输入过长

| 场景 | 处理 |
|------|------|
| 文本超过2000字 | 前端截断并提示"输入过长，已截取前2000字" |
| URL页面内容过大 | 后端提取正文后截断到2000字 |

### 6.4 URL抓取失败

| 场景 | 处理 |
|------|------|
| 目标网站反爬 | 返回错误提示"无法抓取该链接，请尝试直接粘贴文本" |
| 非HTML内容（PDF/视频） | 提示"暂不支持该类型链接，请粘贴文字内容" |
| 链接无效 | 提示"链接无法访问，请检查后重试" |

### 6.5 Groq API额度耗尽

| 场景 | 处理 |
|------|------|
| 429 Rate Limit | 后端catch，前端提示"使用次数已达上限，明日再试" |
| 月度额度接近耗尽 | 后端监控用量，达到80%时限制每用户每日3次 |
