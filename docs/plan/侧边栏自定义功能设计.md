## 一、功能定位

**侧边栏自定义是观微的个性化导航功能——用户可在设置页增删侧边栏导航项，选择自己常用的模块放在侧边栏，提升使用效率。持久化到localStorage，保底逻辑确保至少保留1个导航项。**

解决的问题：
1. **导航项过多**——默认6项导航，部分用户只需要其中几项
2. **常用模块不同**——创作者常用工作间，消费者常用瓜田，需求各异
3. **个性化缺失**——固定导航无法适应不同用户习惯

给谁用：所有注册用户。

### 2.4 保底逻辑

```
保底规则：
1. 至少保留1个导航项
2. 用户尝试删除最后一个导航项时，拒绝操作
3. 提示"至少需要保留1个导航项"
4. 如果localStorage数据异常（空数组/null），自动恢复默认导航项

恢复默认：
  - 用户点击"恢复默认"按钮 → 恢复6个默认导航项
  - localStorage损坏 → 读取失败时自动恢复默认
```

### 3.2 sidebarStore 设计

```typescript
// stores/sidebarStore.ts

import { create } from 'zustand'
import { persist } from 'zustand/middleware'

interface NavItem {
  id: string
  label: string
  path: string
  icon: string
  isVisible: boolean       // 是否在侧边栏显示
  isDefault: boolean      // 是否默认导航项
  adminOnly?: boolean      // 是否仅管理员可见
}

interface SidebarState {
  navItems: NavItem[]

  // 操作
  addNavItem: (id: string) => void
  removeNavItem: (id: string) => void
  resetToDefault: () => void
  getVisibleNavItems: () => NavItem[]
}

// persist 中间件自动同步到 localStorage
export const useSidebarStore = create<SidebarState>()(
  persist(
    (set, get) => ({
      navItems: DEFAULT_NAV_ITEMS,

      addNavItem: (id) => set((state) => ({
        navItems: state.navItems.map(item =>
          item.id === id ? { ...item, isVisible: true } : item
        )
      })),

      removeNavItem: (id) => {
        const state = get()
        const visibleCount = state.navItems.filter(i => i.isVisible).length

        // 保底逻辑：至少保留1个
        if (visibleCount <= 1) {
          console.warn('至少需要保留1个导航项')
          return
        }

        set({
          navItems: state.navItems.map(item =>
            item.id === id ? { ...item, isVisible: false } : item
          )
        })
      },

      resetToDefault: () => set({
        navItems: DEFAULT_NAV_ITEMS.map(item => ({
          ...item,
          isVisible: item.isDefault
        }))
      }),

      getVisibleNavItems: () => {
        const state = get()
        const visible = state.navItems.filter(i => i.isVisible)

        // 保底：如果没有任何可见项，返回默认
        if (visible.length === 0) {
          return DEFAULT_NAV_ITEMS.filter(i => i.isDefault)
        }

        return visible
      }
    }),
    {
      name: 'jw_sidebar_custom',  // localStorage key
      version: 1,
    }
  )
)
```

### 3.4 SidebarNav 渲染逻辑

```typescript
// components/sidebar/SidebarNav.tsx

function SidebarNav() {
  const { getVisibleNavItems } = useSidebarStore()
  const { user } = useAuthStore()

  const visibleItems = getVisibleNavItems().filter(item => {
    // 管理员专属项过滤
    if (item.adminOnly && !user?.isAdmin) return false
    return true
  })

  return (
    <nav>
      {visibleItems.map(item => (
        <NavLink key={item.id} to={item.path}>
          <Icon name={item.icon} />
          <span>{item.label}</span>
        </NavLink>
      ))}
    </nav>
  )
}
```

## 四、异常场景应对

### 4.1 localStorage 读取失败

```
场景：隐私模式或localStorage已满
处理：
  - persist 中间件自动降级为内存模式
  - 侧边栏显示默认导航项
  - 不阻塞用户使用
  - 控制台输出警告但不崩溃
```

### 4.2 数据损坏

```
场景：navItems 数组为空或格式错误
处理：
  - getVisibleNavItems() 检测到 visible.length === 0
  - 返回 DEFAULT_NAV_ITEMS 中 isDefault=true 的项
  - 用户侧无感知，侧边栏显示默认项
```

### 4.3 管理员权限变化

```
场景：普通用户添加了"管理后台"导航项，后来被取消管理员权限
处理：
  - SidebarNav 渲染时检查 adminOnly + user.isAdmin
  - 非管理员不显示 adminOnly 项
  - 设置页中对非管理员隐藏"管理后台"添加选项
```

### 4.4 路由不存在

```
场景：导航项指向的路由已被移除（如情绪检测 /tools/emotion 已砍掉）
处理：
  - sidebarConfig 中不包含已移除的路由
  - 如果用户localStorage中有已移除的路由，渲染时过滤掉
  - 不影响其他导航项
```

## 六、与其他模块的联动

```
侧边栏自定义 → 设置页
  在设置页提供"侧边栏自定义"区域
  操作后实时生效

侧边栏自定义 → 全局侧边栏
  SidebarNav 读取 sidebarStore 渲染
  所有使用 layout（带侧边栏）的页面都受影响

侧边栏自定义 → 用户体系
  管理员专属导航项（管理后台）需要检查 isAdmin
  非管理员不可见

侧边栏自定义 → localStorage
  persist 中间件自动同步
  key: 'jw_sidebar_custom'
```
