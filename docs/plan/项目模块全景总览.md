# 观微项目 — 业务模块全景总览

> 版本：v3.1（新增AI创作助手模块）
> 基于 `src/` 全量源码审计 + 架构审查报告 + 产品方向校准
> 核心原则：以"AI辅助内容质量社区"为定位，AI创作助手是供给侧引擎，瓜田+AI竞技场是消费侧双引擎

## 一、优先级重新定义

### 1.1 优先级判定标准

| 等级 | 含义 | 判定标准 |
|------|------|---------|
| **P0-必须完成** | hackathon演示的核心闭环，缺了产品不成立 | 用户进来看不到东西 / 无法完成核心交互 |
| **P1-尽量完成** | 提升演示质量，缺了也能demo | 功能可用但体验粗糙 |
| **P2-可以demo** | 有页面能点就行，不要求功能完整 | 展示产品广度，不要求深度 |
| **P3-砍掉** | hackathon前不开发 | 开发量大、收益低 |

### 1.2 模块优先级表

| 模块 | 优先级 | 判定理由 |
|------|--------|---------|
| **AI创作助手** | **P0-产品活路** | 供给侧引擎，解决"内容从哪来"的根本问题。降低创作门槛=让更多人产出高质量内容 |
| **瓜田系统** | P0-必须完成 | 消费侧核心，用户进来的第一件事就是刷瓜。AI创作助手产出的内容在此消费 |
| **AI竞技场** | P0-必须完成 | 病毒传播引擎，零参与门槛，不依赖内容供给 |
| **用户体系** | P0-必须完成 | 段位+积分是留存基础设施，但只做简化版 |
| **佐证系统** | P0-必须完成 | AI创作助手的下游产出目标之一（创作→发布为佐证） |
| **管理后台** | P0-必须完成 | 已有基础实现，补充运营功能即可 |
| **热点系统** | P1-尽量完成 | 流量入口，但手动录入即可 |
| **求证工具集** | P1-尽量完成 | AI创作助手的事实核查功能依赖其LLM求证能力 |
| **辩论系统** | P1-尽量完成 | 只做AI斗蛐蛐（已有基础），真人辩论demo模式 |
| **社区系统** | P2-可以demo | AI创作助手的另一个下游产出目标 |
| **笔记系统** | P2-可以demo | 简单文本编辑器+发布为佐证 |
| **LLM配置** | P2-可以demo | 页面已有，保持现状。AI创作助手是其第一个真正全局联动的消费者 |
| **平台架构** | P0-基础设施 | 已稳定，维护即可 |

## 二、MVP范围定义

### 2.1 hackathon演示的核心闭环

```
                    ┌── 消费侧 ──┐
                    │              │
用户进入 → 看到瓜田（有内容） → 猜真假 → 看开奖
                ↑                    ↓
         ┌──────┘              写高质量佐证
         │                      （AI辅助）
    AI创作助手                       │
    （供给侧引擎）                    ↓
    降低创作门槛              佐证/帖子沉淀在瓜田
    学习用户风格                    │
    事实核查                       ↓
                            看AI斗蛐蛐 → 下注
                                         ↓
                                   段位+积分（成就感）
```

**核心叙事**：AI帮你写出有信息密度的内容 → 内容留在社区被消费 → 社区有留存 → 更多内容 → 正循环

### 2.2 每个模块的MVP功能清单

#### 瓜田系统（P0）

| 功能 | MVP要求 | 不做 |
|------|---------|------|
| 瓜列表 | 瀑布流，有10+种子瓜 | 推荐流/协同过滤 |
| 瓜详情 | 展示信息+猜真假+佐证列表 | 实锤报告完整版 |
| 猜瓜 | 选真/假，记录选择 | 赔率计算 |
| 佐证 | 纯文字佐证，按方向分组，投票 | AI质量评分/图片佐证 |
| 开奖 | 管理员在后台手动开奖 | AI自动开奖 |
| 内容供给 | LLM生成50-100条种子瓜 | 自动爬取/用户自发瓜 |

#### AI竞技场（P0）

| 功能 | MVP要求 | 不做 |
|------|---------|------|
| AI角色 | 6个预设角色，各有性格prompt | 用户自定义角色 |
| 自动辩论 | AI双方自动轮流发言，5-8轮 | 自定义轮数 |
| 裁判评分 | 辩论结束后AI裁判给出胜负 | 实时胜率分析 |
| 弹幕 | 观众发弹幕 | 弹幕防刷/弹幕礼物 |
| 围观下注 | 简单的选边下注 | 赔率系统/积分结算 |

#### 用户体系（P0-简化版）

| 功能 | MVP要求 | 不做 |
|------|---------|------|
| 段位 | 7段位，基于猜对数+准确率晋升 | 保级/降级 |
| 积分 | 猜瓜+佐证+签到获取 | 12种来源只做3种 |
| 个人主页 | 基本信息+段位+积分 | Notion式自定义 |
| 徽章 | 不做 | 15个徽章全砍 |

#### 佐证系统（P0）

| 功能 | MVP要求 | 不做 |
|------|---------|------|

| 质量评分 | 发布前4维度评分+改进建议 | 阻止低质量发布 |
| 一键发布 | 发布到瓜田佐证/社区帖子 | 多平台同步发布 |
| LLM统一入口 | 重构4个service使用callLLM() | 完整provider抽象层 |

## 三、跨模块依赖（具体实现方式）

不是抽象的箭头图，而是具体的代码级依赖：

### 3.1 瓜田 → 佐证

```
verificationStore.ts:
  - getMelonDetail(melonId) 返回的 melon.evidences 来自 evidenceStore
  - 实际调用: evidenceStore.getEvidencesByMelonId(melonId)
  - 数据流: evidenceStore.evidences[] → 按 melonId 过滤 → 按 direction 分组
```

### 3.2 瓜田 → 用户

```
verificationStore.ts → userStore.ts:
  - 猜瓜成功后: userStore.addPoints(amount) // 积分+10
  - 猜瓜后更新准确率: userStore.updateAccuracy(correct, total)
  - 段位自动计算: userStore.recalculateRank() // 基于accuracy和totalGuesses
```

### 3.3 瓜田 → 社区

```
MelonDetailPage.tsx:
  - 瓜数据中有 communityId 字段
  - "去社区讨论"按钮 → navigate(`/community/${melon.communityId}`)
  - communityStore.getCommunity(communityId) 获取社区名称和帖子数
```

### 3.4 AI竞技场 → LLM

```
aiBattleService.ts → llmService:
  - 每轮辩论发言: llmService.chat(rolePrompt, context) 
  - 裁判评分: llmService.chat(judgePrompt, debateHistory)
  - 降级: llmService.isAvailable() === false → 显示预设台词
```

### 3.5 笔记 → 佐证

```
NoteDetailPage.tsx → evidenceStore:
  - 发布为佐证: evidenceStore.submitEvidence({ melonId, content: note.content, direction })
  - 佐证来源标记: evidence.source = 'note', evidence.sourceNoteId = note.id
```

### 3.6 管理后台 → 全部模块

```
AdminPage.tsx:
  - 用户管理 → userStore.getUsers() / userStore.banUser(id)
  - 瓜管理 → verificationStore.getMelons() / verificationStore.setMelonResult(id, result)
  - 审核 → contentReviewService.review(contentId, action)
  - 数据看板 → analyticsService.getDashboard()
```

### 3.7 AI创作助手 → 多模块（供给侧引擎）

```
creationService.ts → llmStore.callLLM():
  - 骨架生成: callLLM(skeletonPrompt, { topic, stance, readerType })
  - 质量评分: callLLM(scoringPrompt, { draft, styleProfile })
  - 风格分析: callLLM(stylePrompt, { recentPosts })

creationService.ts → verificationStore:
  - 事实核查: verificationStore.verify(claim) → 复用求证工具的LLM文字求证
  - 核查结果内联在编辑器中

creationStore.ts → evidenceStore:
  - 发布为佐证: evidenceStore.submitEvidence({ melonId, content, direction, source: 'ai-assisted' })
  - 佐证来源标记: evidence.aiAssisted = true

creationStore.ts → communityStore (v2):
  - 发布为帖子: communityStore.createPost({ communityId, content, source: 'ai-assisted' })

creationService.ts → userStore:
  - 发布奖励: userStore.addPoints(5) // 创作发布+5（高于普通佐证+3）
  - 高质量奖励: 质量评分>80 → userStore.addPoints(3)

llmStore.callLLM() 重构:
  - aiJudgeService → callLLM()       // 原hardcode Groq
  - aiSummaryService → callLLM()     // 原hardcode Groq
  - aiTranslatorService → callLLM()  // 原hardcode Groq
  - emotionAnalysis → callLLM()      // 原hardcode Groq
  - creationService → callLLM()      // 新模块，直接使用统一入口
  - styleProfileService → callLLM()  // 新模块
```

### 3.8 AI创作助手 → Mirror引擎（v2路径）

```
Phase 1（MVP）: 前端直调LLM
  creationService.ts → llmStore.callLLM() → Groq API

Phase 2（Mirror集成）:
  creationService.ts → Mirror Gateway API (port 8000)
    → Commander（意图分析 → 任务分解）
      → Researcher Agent（搜索资料，调用abilities工具集）
      → Outliner Agent（生成骨架）
      → Style Agent（风格适配）
    → AgentPool热替换兜底（超时/失败自动换Agent）
    → WorldBridge WebSocket推送实时状态到前端
    → MyDesignBackend存储用户style profile（跨会话持久化）
```

| 风格档案生效 | 选择风格→生成骨架→对比不同风格骨架差异 | 不同风格生成的骨架结构和语气不同 |
| AI斗蛐蛐观赏 | 进入AI竞技场→选角色→观看辩论→发弹幕→下注 | 辩论自动完成，弹幕可见，积分扣除 |
| 笔记→佐证 | 写笔记→发布为佐证→选择瓜→佐证出现 | 佐证出现在瓜详情页 |
| 管理员操作 | 登录后台→审核内容→开奖→紧急下线 | 操作生效，前端状态同步 |
| LLM降级 | 开启降级开关→尝试AI竞技场→尝试求证→尝试创作助手 | 各模块按策略降级，提示友好 |
| LLM配置切换 | 设置页切换provider→创作助手生成骨架→AI竞技场辩论 | 全部使用新provider，无hardcode残留 |

## 七、数据层汇总

| Store | 内容 | 持久化 | 依赖模块 |
|-------|------|--------|---------|
| `useUserStore` | 用户信息、积分、段位 | 内存（后端就绪后切API） | 瓜田、佐证、AI竞技场、创作助手 |
| `useVerificationStore` | 瓜列表、瓜详情、猜瓜记录 | 内存 | 佐证、社区、热点、创作助手（事实核查） |
| `useEvidenceStore` | 佐证列表、投票状态 | 内存 | 瓜田、笔记、创作助手（发布目标） |
| `useDebateStore` | 辩论房间、发言、评分 | 内存 | AI竞技场 |
| `useLLMStore` | LLM配置 | Zustand + localStorage | **全部AI模块**（重构后全局联动） |
| `useAuthStore` | 认证Token | 内存 | 全部模块 |
| `useCreationStore`（新增） | 当前草稿、骨架、风格档案 | 内存 + localStorage（草稿自动保存） | 瓜田、佐证、社区、求证工具 |
| `useCommunityStore`（新增） | 社区列表、帖子、评论 | 内存 | 瓜田 |
| `useNoteStore`（新增） | 笔记列表 | 内存 + localStorage | 佐证 |
