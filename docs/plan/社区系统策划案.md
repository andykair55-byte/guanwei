# 见微「社区系统」精简策划案

> 版本：v3.0（MVP精简版）
> 状态：待开发（P2）
> 定位：内容讨论的壳，依附于瓜田/辩论等内容实体

---

## 一、MVP范围

### 做什么

- 官方创建 **10个公开领域社区**（科技/娱乐/社会/财经/体育/游戏/校园/职场/情感/生活）
- 帖子列表（热门/最新/精华三种排序）
- 发帖（标题+正文）
- 评论（单层，不做楼中楼）
- 瓜详情页→"去社区讨论"入口

### 不做什么

| 砍掉的功能 | 砍掉理由 |
|-----------|---------|
| 封闭社区/校园社区/活动社区 | MVP只需公开社区，用户量不够支撑分组 |
| 管理员审批加入 | 公开社区无需审批 |
| 角色权限（创建者/管理员/成员） | 社区由官方创建，不需要用户管理权限 |
| 帖子类型（melon/discussion/event/debate） | 只做普通帖子，不区分类型 |
| 禁言/踢人/置顶 | 管理操作由后台AdminPage统一处理 |
| 用户创建社区 | 官方预置，不开放自建 |

## 二、数据模型（精简版）

```typescript
interface Community {
  id: string
  name: string           // "科技真相"
  description: string    // "关注科技圈最新动态"
  icon: string           // emoji或图标名
  memberCount: number    // 关注人数
  postCount: number      // 帖子数
  createdAt: string
}

interface CommunityPost {
  id: string
  communityId: string
  userId: string
  title: string
  content: string        // 纯文本，支持换行
  isPinned: boolean      // 管理员置顶（后台操作）
  isFeatured: boolean    // 精华帖（后台标记）
  likeCount: number
  commentCount: number
  createdAt: string
}

interface CommunityComment {
  id: string
  postId: string
  userId: string
  content: string
  likeCount: number
  createdAt: string
}
```

## 四、与瓜田联动

**瓜详情页 → 社区入口**

在 `MelonDetailPage` 底部增加一个入口卡片：

```
┌──────────────────────────────┐
│ 💬 去社区讨论                  │
│ 已有 89 人在「科技真相」讨论   │
│ → 查看讨论                    │
└──────────────────────────────┘
```

点击后跳转至对应社区详情页。关联逻辑：每个瓜有一个 `communityId` 字段，指向所属社区。如果瓜没有关联社区，则不显示此入口。

**实现方式**：`verificationStore` 中瓜的 `communityId` 字段 → 路由跳转 `/community/{communityId}`。

## 五、冷启动方案

### 5.1 种子帖子

每个社区预置 **10-20条种子帖子**，由LLM批量生成：

```typescript
// 种子帖子生成规则
const seedPostConfig = {
  communityId: 'tech',
  count: 15,
  templates: [
    '如何看待{event}？',
    '{claim}到底是真是假？',
    '有没有人验证过{claim}？',
    '关于{topic}，我发现了一个有趣的线索...',
  ]
}
```

生成后由管理员在后台审核，确保内容质量。

### 5.2 种子帖子内容要求

- 标题必须有吸引力（问句/悬念）
- 正文100-300字，不能太短
- 模拟真实用户语气，不能有明显AI痕迹
- 分布在不同时间段（不要全是"刚刚发布"）

## 六、异常场景处理

### 6.1 空社区

如果某社区帖子数为0（极端情况，种子帖子被大量删除）：

```
┌──────────────────────────────┐
│ 这个社区还没有帖子             │
│ 成为第一个发帖的人吧！         │
│                              │
│ [📝 发第一条帖子]             │
└──────────────────────────────┘
```

### 6.2 帖子被删除

- 用户访问已删除帖子的URL → 重定向回社区详情页，toast提示"该帖子已被删除"
- 评论列表中已删除帖子的评论 → 不展示（帖子不存在则评论不可见）
- 管理后台删除帖子 → 帖子软删除（标记 `isDeleted: true`），不物理删除

### 6.3 发帖/评论失败

- 网络错误 → 本地缓存草稿（localStorage），提示"发送失败，已保存草稿"
- 内容违规 → 前端关键词预检 + 后端审核，拒绝时提示具体原因
- 重复提交 → 前端防抖（500ms）+ 后端幂等校验

## 八、验收标准

1. 10个社区正常展示，每个社区有10+种子帖子
2. 帖子列表支持热门/最新/精华三种排序切换
3. 发帖→帖子出现在列表中，评论数+1
4. 评论→评论出现在帖子下方
5. 瓜详情页→"去社区讨论"可正确跳转
6. 空社区有引导UI，删除帖子有正确提示
