# 见微「笔记系统」精简策划案

> 版本：v3.0（MVP精简版）
> 状态：待开发（P2）
> 定位：轻量文本记录工具，可一键发布为佐证

---

## 一、MVP范围

### 做什么

- 简单文本编辑器：标题 + 正文(textarea) + 标签
- 笔记列表页（最近/标签筛选）
- 笔记详情页（查看/编辑）
- **核心联动**：笔记可一键发布为佐证

### 不做什么

| 砍掉的功能 | 砍掉理由 |
|-----------|---------|
| Notion式块编辑器 | 开发量巨大（至少2周），MVP不需要 |
| 动态引用（瓜引用/辩论引用/佐证引用） | 引用链维护复杂，MVP阶段无必要 |
| 图文混排 | 图片上传增加后端复杂度 |
| 笔记分享/公开 | MVP阶段笔记是纯私有的 |
| 笔记模板 | 过度设计 |
| 导出为辩论陈词 | 辩论系统本身降级为demo |

## 二、数据模型（精简版）

```typescript
interface Note {
  id: string
  userId: string
  title: string
  content: string          // 纯文本，textarea内容
  tags: string[]           // 标签，最多5个
  relatedMelonId?: string  // 关联的瓜ID（可选）
  createdAt: string
  updatedAt: string
}
```

对比原方案砍掉了：`NoteBlock[]`（块内容）、`referenceMelons`、`referenceEvidences`、`referenceDebates`、`likeCount`、`collectCount`、`isPinned`。

## 四、核心联动：笔记→佐证

这是笔记系统唯一的跨模块价值点，必须做好。

### 4.1 交互流程

```
笔记编辑页/查看页
  → 点击「发布为佐证」
  → 弹窗选择目标瓜（如果笔记没有关联瓜）
  → 弹窗选择佐证方向：支持真 / 支持假
  → 确认发布
  → 笔记内容作为佐证文本，写入 evidenceStore
  → 跳转到瓜详情页的佐证列表
```

### 4.2 实现方式

```typescript
// 在 NoteDetailPage 中
const publishAsEvidence = async (note: Note) => {
  // 1. 选择目标瓜
  const targetMelon = await selectMelon(note.relatedMelonId)
  if (!targetMelon) return

  // 2. 选择方向
  const direction = await selectDirection() // 'true' | 'false'

  // 3. 调用佐证服务
  await evidenceStore.submitEvidence({
    melonId: targetMelon.id,
    content: note.content,
    direction,
    source: 'note',        // 标记来源为笔记
    sourceNoteId: note.id, // 关联笔记ID
  })

  // 4. 跳转
  navigate(`/melon/${targetMelon.id}`)
}
```

### 4.3 约束

- 同一篇笔记只能发布为一条佐证（防止重复）
- 发布后笔记内容修改不会同步更新佐证（佐证是快照）
- 发布佐证获得积分奖励（与直接写佐证相同）

## 五、异常场景处理

### 5.1 编辑中断

- 自动保存到 localStorage（每30秒），key: `note_draft_{noteId}`
- 下次打开时检测草稿，提示"恢复上次未保存的内容？"
- 手动保存成功后清除草稿

### 5.2 标题/正文为空

- 标题为空 → 发布按钮禁用，提示"请输入标题"
- 正文为空 → 允许保存（可以是短笔记），但发布为佐证时要求正文≥20字

### 5.3 标签管理

- 最多5个标签，每个标签≤10字
- 输入标签时自动补全已有标签（从用户历史标签中匹配）
- 标签只用于列表页筛选，不做全局标签聚合页

## 七、验收标准

1. 创建笔记：标题+正文+标签，保存成功
2. 列表页按最近/标签筛选，排序正确
3. 编辑页自动保存草稿，中断后可恢复
4. 笔记→发布为佐证：选择瓜+方向后，佐证出现在瓜详情页
5. 同一篇笔记不可重复发布为佐证
6. 空笔记列表有引导提示
