## 总体节奏

```
Phase 1: 前端界面重构（Sprint 1-2，10个工作日）
  目标：所有P0模块UI到位，每个页面都能走通mock流程
  方法：参照策划案逐个模块改造现有页面，新增工作间/trae宝/娱乐大厅页面
  不碰：后端API、LLM调用逻辑

Phase 2: AI能力集成（Sprint 3，5个工作日）
  目标：callLLM统一入口重构完成，工作间/trae宝/娱乐大厅核心AI功能可用
  方法：重构4个hardcode Groq的service → callLLM()，接入工作间/trae宝AI功能
  依赖：Phase 1的UI组件已就位

Phase 3: 后端稳定化（Sprint 4-5，10个工作日）
  目标：后端API逐模块对接，Agent容错机制验证
  方法：围绕AI稳定性，逐个API端点从mock切换到真实后端
  依赖：Phase 2的AI调用链路已验证
```

**架构审查新增模块排期**：
- 工作间（原AI创作助手合并工具）：Sprint 2 搭建，Phase 2 接入AI
- trae宝（全局聊天agent）：Sprint 2 搭建，Phase 2 接入AI
- 娱乐大厅（聚合人机对决/AI斗蛐蛐）：Sprint 1-2 改造现有AI竞技场
- 侧边栏自定义：Sprint 2 搭建

