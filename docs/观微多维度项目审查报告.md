## 项目全景速写

观微是一个信息求证社区 App，核心玩法是让用户对热点事件"猜真假"，辅以 AI 工具链帮助求证。技术栈 React 19 + Vite 8 + TypeScript + TailwindCSS v4 + Zustand，后端 FastAPI + LangGraph 多 Agent 管线 + SQLite（待迁 PostgreSQL）。PC 端 Twitter 式三栏布局，移动端 DeviceFrame 真机模拟。正参加 adventure-x.org 黑客松社会服务赛道。

产品功能矩阵：瓜田（猜真假社区）、求证工具箱（EXIF 鉴定 / 情绪操控检测 / 反向搜图 / 时间线重建 / 洗稿检测 / 多源验证 / 一键求证）、辩论场（3v3 真人辩论 + AI 评判）、社区论坛、积分段位系统、管理后台。

## 第一视角：用户

> 角色设定：一个 25 岁的普通互联网用户，日常刷微博和小红书，偶然看到朋友转发了观微的链接，好奇点进来看看。

### 第一印象（前 30 秒）

打开首页，瓜田的卡片流视觉冲击力不错——封面图、分类标签、真假投票比例一目了然。但问题立刻出现：**所有瓜都是 mock 数据，标题有"某顶流男星"这种一看就是占位符的内容**。用户的第一反应不是"这个产品好酷"，而是"这是个 demo 吧"。

社区页面有 22 条 mock 帖子，但内容同质化严重，缺少真实用户的讨论氛围。对于一个社区产品来说，**冷启动阶段的内容质感决定了用户是否愿意留下来**。

### 核心体验链路分析

| 步骤 | 用户行为 | 体验评价 | 问题 |
|------|----------|----------|------|
| 1. 浏览瓜田 | 看到瓜 → 想参与猜测 | 视觉好，卡片信息密度合理 | 瓜的内容虚假感强，无法建立信任 |
| 2. 猜测投票 | 选真或假 → 提交佐证 | 交互流畅 | 未登录时体验断裂，缺少"先体验再注册"的钩子 |
| 3. 查看佐证 | 看别人的佐证和讨论 | 佐证投票机制有趣 | mock 数据下佐证内容为空或假 |
| 4. 等待揭晓 | 等待开奖看结果 | 设计上有期待感 | 没有推送通知，用户不会主动回来 |
| 5. 工具箱 | 用 EXIF / 情绪检测 | EXIF 真的能用，情绪检测需要 Groq | 7 个工具只有 1-2 个真正可用 |
| 6. 辩论场 | 参与 3v3 辩论 | 规则设计精细（字数阶梯/抬走机制） | 没有真人对手，无法体验 |

### 留存机制审视

**积分段位系统**设计得相当用心——7 个段位从"吃瓜群众"到"见微先知"，每个段位有猜对次数、准确率、参与次数三维门槛。但从用户视角看，这更像是游戏化装饰而非真正的留存驱动力。用户不会因为想从"瓜田新手"升到"鉴瓜学徒"而多猜 10 条瓜——**段位的价值感来自社区认同，而非数字本身**。

**每日签到**给了 10 积分，但积分能兑换什么？目前看不到积分商城或任何消费场景。积分只进不出，等于一个没有出口的水库，用户看不到积累的意义。

**辩论场**是最有社区粘性的设计——3v3 真人辩论、AI 评判、逻辑谬误检测、观众"抬走"投票。但这个功能完全依赖实时多人在线，冷启动阶段几乎不可能凑齐 6 个同时在线的人来辩论。**这是一个需要网络效应才能运转的功能，但产品目前还没有网络效应**。

### 用户痛点共情地图

| 维度 | 内容 |
|------|------|
| **用户说** | "这些瓜是真的吗？""猜完了然后呢？""工具箱为什么点不开？" |
| **用户想** | "这个 app 和辟谣平台有什么区别？""我为什么要在这里花时间而不是去搜一下？" |
| **用户做** | 点进 2-3 个瓜看看，试一下 EXIF 工具，发现其他工具不能用，退出 |
| **用户感** | 好奇 → 疑惑 → 失望（发现是 demo）→ 离开 |
| **痛点** | 内容不真实、工具不可用、没有社交互动、缺少回来的理由 |
| **收益** | 如果内容真实、工具可用，"边玩边学辨别信息"的概念是有吸引力的 |

### 用户视角结论

**核心问题不是功能不够多，而是没有一个功能是"活的"。** 瓜田有数据但假的，辩论有规则但没人的，工具有界面但大部分不能用。一个社区产品的第一要务不是功能完整度，而是**有人在这里**。我宁愿看到一个只有 5 条真瓜但有真实讨论的产品，也不想看到一个 20 条 mock 瓜的精致 demo。

## 第二视角：老板 / 产品经理

> 角色设定：项目负责人，关心投入产出比、团队效率、产品节奏、能不能按时交付比赛成果。

### 产品定义与规划审查

#### 第一性原理拆解

| 层级 | 分析 |
|------|------|
| **物理层（不可变约束）** | 信息真伪判断本质上是人类的认知行为，AI 只能辅助不能替代——这是产品哲学的根基，也是技术边界 |
| **需求层（真实需求）** | 用户不是需要"被教"如何辨别信息，而是需要"参与感"——在过程中证明自己的判断力，获得社区认同 |
| **价值层（本质价值创造）** | 将信息求证过程游戏化，让用户在"玩"中提升信息素养。核心价值不是"准确的答案"，而是"求证的体验" |

这个拆解是清晰的。"AI 辅助，人参与"的核心叙事立得住。但问题在于：**产品目前的实现和这个叙事之间有断层**。叙事说的是"人参与"，但产品里没有任何真实的人参与——没有真实用户、没有真实内容、没有真实互动。

#### 功能分层评估

| 层级 | 功能 | 完成度 | 问题 |
|------|------|--------|------|
| **P0 核心** | 猜瓜（真假投票 + 佐证） | 前端完成，后端可用 | mock 数据，缺少真实瓜源 |
| **P0 核心** | 用户注册/登录 | 完成 | 无第三方登录，门槛偏高 |
| **P0 核心** | 积分段位系统 | 完成 | 积分无消费场景，段位无社区价值 |
| **P1 支撑** | EXIF 鉴定 | 完成且可用 | 唯一真正可用的工具 |
| **P1 支撑** | 情绪操控检测 | 前端完成 | 依赖 Groq API，未集成 |
| **P1 支撑** | 一键求证 | 前端完成，后端管线就绪 | 前后端割裂，后端未部署 |
| **P2 增强** | 辩论场（3v3） | 前端完成，后端部分 | 规则设计精良但无法实际运转 |
| **P2 增强** | 反向搜图 | 有页面 | 未实现 |
| **P2 增强** | 时间线重建 | 有页面 | 未实现 |
| **P3 生态** | 洗稿检测 / ELA 检测 | 有页面占位 | 完全未实现 |
| **P3 生态** | 社区论坛 | 前端完成 | mock 数据，无真实内容 |

#### MVP 范围审视

按照 MVP 划定原则，当前项目存在严重的**功能蔓延**问题。7 个工具箱入口只有 1 个能用，3 种辩论模式（AI 对战 / 圆桌 / 真人辩论大厅）全是空壳，社区论坛是 mock 数据。

**一个合格的 MVP 应该是：砍掉 60% 的页面，让剩下的 40% 全部是活的。** 具体来说：瓜田（真实数据）+ EXIF 工具（已可用）+ 情绪操控检测（接 Groq）+ 一键求证简化版（接 Groq）。4 个真正能用的入口，比 20 个空壳页面有说服力得多。

#### 投入产出比分析

| 模块 | 预估投入 | 比赛加分 | ROI |
|------|----------|----------|-----|
| 后端完整架构（LangGraph + 12 家 LLM + JWT + Redis） | 大量 | 中（评委不一定会看代码） | 低 |
| 辩论场 3 代演进（规则系统 + AI 评判 + 抬走机制） | 大量 | 高（如果能演示） | 中 |
| 7 个工具箱页面 | 中 | 低（6/7 不能用） | 极低 |
| 管理后台（JWT + 审计日志 + 指标监控） | 中 | 低（比赛场景不需要） | 低 |
| 前后端联调 | 中 | 高（让产品从 demo 变成产品） | 高 |
| 内容填充（真实瓜数据） | 低 | 极高（第一印象） | 极高 |

**最大的 ROI 洼地是"内容填充"和"前后端联调"。** 花 1 天时间把 mock 瓜换成有真实细节的内容，效果提升可能超过花 2 周写后端架构。

#### 工程质量与团队效率

后端工程质量已在前文详细审查。总结关键发现：6 个 P0 安全问题、Collector Agent 过滤逻辑反转导致搜集功能基本失效、Moderator 正则全部无效、CORS 完全开放。这些问题在比赛阶段不影响演示（评委不会做安全测试），但如果产品要上线，必须全部修复。

前端代码质量明显高于后端——React 19 + lazy loading + 路由级代码分割、Zustand store 分层合理、TailwindCSS v4 统一样式系统。但存在一个架构问题：**前后端完全割裂**，前端的 API 调用全部有 mock fallback，这意味着"接了后端"和"没接后端"的用户体验差异很小，后端的投入没有被前端充分放大。

### 老板视角结论

**项目最大的风险不是技术做不出来，而是做了太多用不上的东西。** 目前的状态是"技术深度有余，产品完整度不足"。建议立即执行 plan.md 中的优先级排序：先做情绪操控检测（3-5 天）和前端打磨（2-3 天），再联调后端。砍掉所有不能用的工具箱页面入口，砍掉辩论场的三个空壳模式，聚焦到"评委能在 3 分钟内体验到的 4 个真功能"。

## 第三视角：投资人

> 角色设定：早期天使投资人，关注市场空间、竞争壁垒、团队执行力、商业化路径。看的是这个方向值不值得投，这个团队值不值得下注。

### 市场机会评估

#### 行业界定

观微切的是"信息素养教育 + 社区"这个交叉地带。往大了说是"信息真伪鉴别"，往小了说是"以游戏化方式提升公众信息判断力"。

| 维度 | 评估 |
|------|------|
| **TAM（总可及市场）** | 中国网民 10.79 亿（CNNIC 2025），理论上都面临信息真伪问题。但这个 TAM 没有意义——不是所有人都会为此付费或使用专门工具 |
| **SAM（可服务市场）** | 对信息素养有主动需求的群体：媒体从业者、学生、教育工作者、中老年易受骗群体。估算千万级用户规模 |
| **SOM（可获得市场）** | 黑客松 → 种子用户 500-1000 人真实测试 → 如果留存率达标再扩展 |

#### 竞品格局

| 竞品类型 | 代表 | 观微的差异化 |
|----------|------|--------------|
| **官方辟谣平台** | 中国互联网联合辟谣平台、腾讯较真 | 它们是"给答案"，观微是"让你自己找答案"——体验完全不同 |
| **AI 事实核查工具** | Full Fact、ClaimBuster | 它们是专业工具，面向记者/研究者；观微是消费级社区 |
| **知识问答社区** | 知乎、果壳 | 它们是开放讨论，没有游戏化机制和结构化求证流程 |
| **信息素养教育** | 学校课程、公益培训 | 它们是"教育"，观微是"娱乐"——没有人喜欢被教育 |

**差异化定位是有价值的。** "过程即娱乐"这个切入点在竞品格局中确实是空白。但空白的另一个解读是：**可能没有足够大的需求**。用户真的会主动来"玩"信息求证吗？这是核心假设，需要数据验证。

#### 竞品盲区识别

| 盲区 | 描述 | 观微的机会 |
|------|------|------------|
| **体验盲区** | 现有辟谣产品都是"告诉你答案"，没有人做"让你参与求证过程" | 这正是观微的核心叙事 |
| **场景盲区** | 没有人把信息求证做成社交游戏 | 猜瓜 + 段位 + 辩论的组合是独创的 |
| **技术盲区** | 多 Agent 协作做信息求证是新的技术应用方式 | LangGraph 管线 + 12 家 LLM 供应商是技术亮点 |

### 竞争壁垒分析

| 壁垒类型 | 评估 | 说明 |
|----------|------|------|
| **技术壁垒** | 弱 | LangGraph 管线和 LLM 集成是可复制的，不构成真正的技术护城河。任何团队用同样的开源工具链都能在几周内搭建类似系统 |
| **数据壁垒** | 无 | 目前零真实用户数据。即使上线后，信息求证领域的数据积累速度慢（不像推荐系统有海量行为数据） |
| **网络效应** | 潜在有 | 辩论场和社区讨论有网络效应，但需要足够的用户密度才能激活。目前是零 |
| **品牌壁垒** | 无 | 产品尚未上线，品牌认知为零 |
| **内容壁垒** | 潜在有 | 如果积累了高质量的求证案例库（实锤报告 + 证据链），可以形成内容护城河 |
| **转换成本** | 弱 | 用户的积分和段位是唯一粘性，但如果竞品提供更好的体验，迁移成本很低 |

**坦率地说，目前的壁垒接近于零。** 产品的全部价值在于"想法 + 实现"，而这两者都不构成壁垒。真正的壁垒需要在运营中建立：要么是社区氛围和文化（最难复制），要么是独家内容库（实锤案例积累），要么是用户关系网络（辩论对手、关注者）。

### 商业化路径审视

目前项目中没有任何商业化设计的痕迹。没有付费功能、没有广告位、没有增值服务入口。

| 潜在商业模式 | 可行性 | 问题 |
|--------------|--------|------|
| **增值服务**（高级工具 / 更多求证次数） | 中 | 工具箱的使用频率低，用户付费意愿存疑 |
| **教育 B2B**（学校 / 培训机构） | 中 | 符合"信息素养教育"叙事，但销售周期长 |
| **内容授权**（实锤报告 / 求证案例库） | 低 | 需要大量高质量内容积累 |
| **广告** | 低 | 与"反虚假信息"的品牌形象冲突 |
| **政府/媒体合作** | 中 | 政策推动期有机会，但依赖关系资源 |

### 团队与执行力评估

| 维度 | 评估 |
|------|------|
| **技术能力** | 强。一个人完成了前端全套 + 后端全套 + 多 Agent 管线 + 12 家 LLM 集成 + 管理后台 + 测试体系。技术广度和执行力毋庸置疑 |
| **产品判断力** | 中偏上。核心叙事"AI 辅助人参与"是精准的，游戏化设计（段位 / 辩论 / 佐证投票）有创意。但存在明显的"做太多"倾向——功能蔓延说明产品取舍能力需要加强 |
| **运营能力** | 未知。项目目前全部是技术投入，没有看到任何内容运营、用户增长、社区建设的实际行动 |
| **商业敏感度** | 弱。零商业化设计，对"怎么赚钱"这个问题没有思考的痕迹 |

### 关键假设清单（需验证）

| 编号 | 假设 | 验证方法 | 当前状态 |
|------|------|----------|----------|
| H1 | 用户愿意主动参与"猜真假"这种游戏化信息求证 | 500-1000 人种子测试，看次日留存和 7 日留存 | 未验证 |
| H2 | 辩论场能产生足够的社区粘性和内容沉淀 | 10 场真人辩论测试，看观众留存和回放率 | 未验证（且冷启动困难） |
| H3 | AI 工具链（EXIF / 情绪检测 / 多源验证）对用户有实际使用价值 | 工具使用率、重复使用率、分享率 | 部分验证（EXIF 可用） |
| H4 | "过程即娱乐"的叙事能吸引非专业用户 | A/B 测试：观微 vs 传统辟谣平台的用户满意度对比 | 未验证 |
| H5 | 积分段位系统能驱动长期留存 | 30 日留存率、段位分布、高段位用户活跃度 | 未验证 |

### 投资人视角结论

**这个项目的核心资产是创始人的技术执行力和产品叙事的独特性。** 一个人搭出前后端完整架构 + 多 Agent 管线 + 12 家 LLM 集成，这个交付速度本身就是能力证明。"AI 辅助，人参与"的叙事在信息求证赛道中确实是差异化的。

但投资不是投能力，是投回报。目前的三个致命问题：

1. **没有真实用户数据。** 所有关于留存、粘性、社区活力的判断都是假设。黑客松的 500-1000 人测试是第一个验证节点，数据出来之前一切都是纸上谈兵。

2. **没有商业化路径。** 一个信息求证社区怎么赚钱？这个问题在项目里完全没有被触及。"提升全民信息素质"是好的社会价值叙事，但不是一个商业模型。

3. **没有壁垒。** 技术壁垒弱（开源工具链），数据壁垒零，网络效应未激活。如果大厂（字节 / 腾讯）决定做类似产品，观微没有任何防御手段。

**如果我是天使投资人，我不会在这个阶段投钱，但我会关注这个创始人。** 理由是：技术能力和产品直觉是够的，但项目还没有跨过"有人用"这道最基本的门槛。等黑客松测试数据出来，如果次日留存 > 30%、7 日留存 > 15%，值得重新评估。

|------|--------|--------|----------|
| **产品完成度** | "看起来是 demo" | "功能很多但大部分不能用" | "MVP 定义不清晰" |
| **核心体验** | "猜瓜挺有意思，但瓜是假的" | "猜瓜是核心，需要真实数据" | "核心体验未经验证" |
| **工具箱** | "EXIF 很酷，其他打不开" | "做了 7 个只用了 1 个" | "技术展示 > 用户价值" |
| **辩论场** | "想玩但没人" | "规则设计精良但冷启动难" | "有网络效应潜力但无法激活" |
| **社区** | "帖子都是假的" | "内容运营零投入" | "社区是潜在壁垒但目前为零" |
| **技术** | 不关心 | "工程质量有硬伤但架构好" | "执行力强但不构成壁垒" |
| **商业化** | 不关心 | "还没想" | "完全没有" |

### 三个角色达成共识的结论

1. **当务之急是让产品"活"起来**——真实数据、真实用户、真实互动。一切技术投入如果不能服务于这个目标，在当前阶段都是过度建设。

2. **功能蔓延是最大风险**——砍掉 60% 的空壳页面，聚焦 4 个真功能，比什么都做但什么都假的策略有效 10 倍。

3. **黑客松测试是生死关**——500-1000 人的真实测试数据将决定这个项目是"一个优秀的学生项目"还是"一个有潜力的产品"。所有精力应该为这个节点服务。

## 附录 A：2026-07-07 方向修正记录（完整思考链路）

> ⚠️ 本附录记录了从审查报告到方向修正的完整推导过程。读完它，你应该能理解每一步「为什么」，而不是只知道「最后改了什么」。

### 一、审查报告暴露了什么

原三视角审查报告的核心结论有三条：

1. **功能蔓延**——做了太多用不上的东西，7 个工具箱只有 1 个能用，3 种辩论模式全是空壳
2. **产品不「活」**——所有数据是 mock 的，没有任何真实用户互动
3. **MVP 范围不清**——不知道砍什么留什么

这三个结论都是症状。**病因是什么？**

回顾当时的页面结构：

| 页面 | 认知状态 |
|------|----------|
| 瓜田 | 模糊：是猜真假游戏还是信息流？ |
| 社区 | 混乱：7 个 Tab + 7 种渐变背景 + 7 种卡片，没人说清这是干嘛的 |
| 其他页面 | 散落：6 个工具箱藏在个人中心，3 个辩论模式无线索 |

**病因找到了：产品的两个核心页面（瓜田 + 社区）没有清晰的定义边界。** 它们内容重叠、功能打架，导致整个产品结构跟着混乱。

---

### 二、第一次拆解：瓜田 vs 社区是什么

**第一步：问「这两个页面分别做什么？」**

| | 瓜田 | 社区 |
|---|---|---|
| 现状 | 猜真假信息流 + 分类筛选 | 7-Tab 内容流 |
| 内容 | 社会热点瓜 | 帖子 |
| 用户行为 | 猜、投票、佐证 | 刷、点赞、评论 |

**问题：看起来差不多，用户分不清。**

**第二步：尝试用「事件 vs 圈子」区分**

| | 瓜田 | 社区 |
|---|---|---|
| 新定义 | 社会热点（事件驱动） | 文化圈子（身份驱动） |
| 类比 | 全城人来看热闹 | 胡同里的同好 |

这个区分逻辑上是对的，但方向错了——它把社区定义成了「圈子」，但项目冷启动阶段根本没有用户来形成圈子。**产品还没有到做圈子的阶段。**

**第三步：纠正——社区不是圈子，是内容广场**

重新思考：在冷启动阶段，社区应该展示什么？
- 不是用户发的水帖（没人发）
- 不是圈子讨论（没人）
- 而是**用户发布的笔记、分析、整理的证据链**

这样社区的角色就从「社交聊天」变成了「UGC 内容消费」——用户来看别人整理的笔记和分析。这是冷启动阶段能成立的定位。

---

### 三、第二次追问：用户为什么需要这个产品？

瓜田和社区的定义清晰了，但还有一个更大的问题没解决：

**市面上已经有微博、抖音、小红书、贴吧、知乎了，用户为什么要来观微？**

逐一排查：

| 平台 | 满足什么 | 不满足什么 |
|------|----------|-----------|
| 微博 | 实时资讯、热搜 | 看完就忘，无法跟踪事件进展 |
| 抖音 | 碎片化消费、推荐算法 | 被动接收，不能主动追溯 |
| 小红书 | 搜索攻略、测评 | 信息散落，没有结构化时间线 |
| 贴吧 | 圈子讨论 | 没有验证机制 |
| 知乎 | 深度问答 | 围绕问题不是围绕事件 |
| 辟谣平台 | 给结论 | 不让你参与判断过程 |

**发现了一个空白地带：所有平台都在做「此刻给你看什么」，没有人在做「帮你记住和理清」。**

---

### 四、痛点是真实存在的

不是理论分析，是现实中每天都在发生的事：

**痛点①：地方负面事件的「记忆黑洞」**

> 这是最具体的例子，也是最核心的设计来源：
>
> 海南每年旅游季爆出海鲜宰客、出租车绕路等负面新闻→ 上热搜→ 文旅局整改→ 发声明→
> → 半年后：没人记得整改了。游客搜「海南旅游」看到的全是负面新闻。
> → 文旅局花了大量人力整改，信息差让他们的努力白费。
> → 打假博主反复曝光同一类问题，没有工具能让他们看到「上次曝光后到底改了没有」。
>
> **这不是「有没有整改」的问题，是「整改的信息没有被记住」的问题。**

**痛点②：追踪连续剧式事件的碎片化**

> 某顶流「被曝隐婚 → 工作室否认 → 狗仔第二波证据 → 品牌解约 → 本人道歉」
> 跨两周的事件，在抖音上今天刷到否认明天刷到解约，中间发生了什么？接不上。
>
> **用户不是不想知道全貌，是没有工具让他们看到全貌。**

**痛点③：普通人的监督权没有工具支撑**

> 群众有监督权，但监督需要持续关注。一个人不可能每天搜一遍「上次那个事件现在怎么样了」。
> 如果能自动跟踪——事件更新时通知你，时间线自动追加——监督门槛就从「每天手动搜」
> 降到了「订阅一下就行」。

**这三个痛点指向同一个答案：事件时间线收束。**

---

### 五、收束：核心差异的诞生

> **定义：事件时间线收束 = 将一个事件在互联网上散落的碎片自动汇聚成一条结构化的时间线，让用户看到全貌。**

为什么 LLM 天然适合做这个？
- 多源搜集：同一事件的不同媒体报道 → AI 自动合并
- 关键节点提取：时间、人物、事件 → 结构化呈现
- 持续追踪：事件有更新 → 自动追加到已有时间线
- 可视化：时间线展示、证据链 → 直观易懂

后端已有的 LangGraph + 多 Agent 管线可以直接复用。不重新发明轮子。

**这不是一个已有平台在做的原因：**

| 平台 | 为什么不 | 观微为什么可以 |
|------|---------|---------------|
| 微博 | 商业模式是实时热搜，不关心历史 | 没有历史包袱 |
| 抖音 | 推荐算法 KPI 是停留时长 | 不做推荐，做主动跟踪 |
| 小红书 | 笔记靠人写 | Agent 自动整理初稿 |
| 辟谣平台 | 只给结论不给过程 | 过程即娱乐 |

### 六、三个维度各自闭合

#### 6.1 逻辑闭环

```
LLM 多源抓取热点 → 事件时间线 → 用户看到 + 猜真假
       ↓
用户用 Agent 辅助整理自己的证据笔记
       ↓
  发布到社区 ← → 自己保留
       ↓
社区内容反哺 → 事件时间线更丰富 → 回到起点
```

每一环的输出是下一环的输入。

#### 6.2 用户行为闭环

```
每天的理由：
  热点时间线每天更新 → 回来看看进展
  社区有新的分析笔记 → 有新鲜内容
  瓜田猜真假有 deadline → 错过就没了

留下来的理由：
  关注的事件自动追踪 → 不需自己记
  你整理的笔记有人看 → 成就感
  段位在涨 → 成长感

带朋友来的理由：
  「这条瓜你猜对了没？我猜对了哈哈哈」→ 社交传播
  朋友可以来看你整理的笔记 → 你的展示
  多人协作笔记 → 像 Notion 但带 AI
```

#### 6.3 商业价值闭环

```
冷启动：事件时间线收束 → 没有竞品 → 差异化吸引
留存：用户跟踪的事件每天更新 → 回来查看 → 行为绑定
壁垒：时间线库 → 独家内容资产 → 网络效应
```

### 七、更新后的竞品格局

```
                    信息深度
                       ↑
               【观微】事件全貌
                       │
   小红书 ──── 知乎 ──── 微博 ──── 抖音
   笔记       问答       实时       碎片
                       │
                 辟谣平台
                 只给结论
                       ↓
                 功能型
```

观微不跟微博比快，不跟抖音比爽，不跟知乎比深。
比的是：**几分钟内看全一个事件的完整生命周期。**

