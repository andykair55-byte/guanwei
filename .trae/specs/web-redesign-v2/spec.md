# 观微 Web 端 UI 重构 v2 - Product Requirement Document

## Overview
- **Summary**: 按照新设计稿重构观微 Web 端的整体布局和首页信息流，从当前的 Zine 杂志风切换为现代克制的内容社区风格（参考小红书 Web + Twitter/X 的简洁感），解决"今日头条式强热点轰炸"的视觉问题，采用经典三栏布局 + 全局顶栏的标准 Web 产品架构。
- **Purpose**: 当前 UI 存在栏目强热点、信息密度过高、导航逻辑混乱（瓜田/社区/发帖并列）的问题，新设计让信息层级更清晰、视觉更年轻、更符合大学生用户审美。
- **Target Users**: 大学生用户群体，偏好简洁现代、信息层级分明的内容消费体验。

## Goals
- 实现全局顶部导航栏（搜索框居中 + 发布按钮 + 通知 + 用户头像）
- 重构左侧导航栏为内容导向菜单（推荐/关注/求证/辩论场/吃瓜榜/公益/话题/收藏/历史/草稿箱）+ 数据统计卡片
- 重构右侧面板为四模块（正在激辩/吃瓜榜/热门话题/推荐用户）
- 重构首页为双列瀑布流布局，顶部带 Hero Banner 和双层分类 Tab
- 卡片组件使用左上角状态角标（🔥热门/✅已求证/🟡求证中/💬辩论中），克制不抢眼
- 左下角增加产品数据展示（今日已求证条数、AI 正在求证话题数），增强信任感

## Non-Goals (Out of Scope)
- 不修改移动端布局和组件
- 不修改后端 API 接口
- 不重构详情页（MelonDetailPage/CommunityDetailPage）的内部布局
- 不修改辩论场、求证页、个人中心等非首页页面的内容结构（仅适配新全局布局）
- 不引入新的第三方 UI 库
- 不改变现有路由结构，首页默认仍为 /community（但其内容重构为"推荐"综合信息流）

## Background & Context
- 当前项目使用 React + TypeScript + Tailwind CSS + Vite
- 已有完整的设计系统 v4（颜色、字体、阴影、动画工具类）在 [index.css](file:///D:/code/code/program/4-%E8%A7%82%E5%BE%AE/src/index.css) 中定义
- 品牌色为朱砂红 `#c0392b`（seal），文字使用 ink-100~900 灰阶
- 字体系统：Noto Serif SC（标题）+ Noto Sans SC（正文）+ JetBrains Mono（数字）
- 左侧栏已有展开/收起功能，右侧栏已有隐藏/展开功能，这些交互保留
- 现有卡片组件 [MelonCard.tsx](file:///D:/code/code/program/4-%E8%A7%82%E5%BE%AE/src/components/MelonCard.tsx) 和 [CommunityCard.tsx](file:///D:/code/code/program/4-%E8%A7%82%E5%BE%AE/src/components/CommunityCard.tsx) 为列表式/大图式，需要改为统一的双列卡片式

## Functional Requirements

### FR-1: 全局顶部导航栏
- 固定在页面顶部，高度约 56-64px
- 左侧：观微 Logo（黑橙图标 + "观微"文字），点击回到首页
- 中间：搜索框，占位文字"搜索帖子、话题、用户..."，圆角胶囊样式
- 右侧：发布按钮（黑底白字圆角按钮，带羽毛笔图标）、通知铃铛图标、用户头像（圆形，点击进入个人中心）

### FR-2: 左侧导航栏重构
- 宽度约 200px（收起态约 64px，保留现有收起/展开交互）
- 菜单项（从上到下）：推荐（⭐星图标，默认选中）、关注（👥人图标）、求证（🔍放大镜图标）、辩论场（⚔️交叉剑图标）、吃瓜榜（🔥火焰图标）、公益（💚心形图标）、话题（# 号图标）
- 分隔线后：我的收藏、浏览历史、草稿箱
- 底部：数据统计卡片
  - "今日已求证" + 大号数字（如 128）+ "较昨日 +23" + 小红线趋势图
  - "AI 正在求证" + 大号数字（如 12）+ "个话题" + 小机器人图标
- 最底部版权：观微·2024 / AI辅助求证社区

### FR-3: 右侧面板重构
- 宽度约 280-300px
- 模块顺序：
  1. **正在激辩**：带编号排名，显示辩题、参与人数（如 1.2w 人参与），前 3 名编号用红色
  2. **吃瓜榜**：带编号排名，显示标题、热度图标和数字，第一名带 🔥 热 标签
  3. **热门话题**：# 话题标签 + 讨论数（如 12.3w 讨论）
  4. **推荐用户**：头像 + 昵称 + 身份标签（求证达人/辩论高手/科学科普博主）+ 红色"关注"按钮
- 每个模块右上角有"更多 >"链接
- 顶部保留收起按钮（设计稿中未画出，但现有交互保留以给用户更大阅读空间）

### FR-4: 首页（推荐页）内容重构
- 路由仍为 `/community`，作为默认首页
- 顶部一级分类 Tab（横排，下划线指示器）：推荐（红色激活）、热门、求证、辩论、公益、校园、生活
- Hero Banner 区域：渐变紫蓝色背景，左侧文案"观微·让信息更透明"+ "AI辅助求证，和大家一起看清真相" + "了解观微"按钮；右侧 3D 放大镜/柱状图插画（可用现有 hero.png 或纯 CSS 渐变块替代）
- Banner 下方二级分类 Tab（灰色胶囊按钮）：全部、娱乐吃瓜、社会热点、校园趣事、科技数码、生活分享，右侧带下拉箭头
- 内容区：双列瀑布流布局（等宽两列，按序排列，左右交替），卡片之间间距适中

### FR-5: 统一内容卡片组件（HomeCard）
- 图片在上，高度自然变化（150-280px 区间），不强制等比例
- 图片左上角：状态标签（彩色圆角小标签，白色文字）
  - 🔥 热门 → 红/橙色底
  - ✅ 已求证 → 绿色底
  - 🟡 求证中 → 橙黄色底
  - 💬 辩论中 → 紫色底
- 图片右上角：时间显示（如 "1小时前"，白色半透明圆角标签）
- 图片下方：标题（2行截断，深色文字，font-weight 600）
- 标题下方：分类标签（浅色圆角小标签，如"娱乐吃瓜"淡紫底/淡色底）+ 作者头像+昵称 + 右侧心形点赞图标+数字

### FR-6: 交互与动效
- 保留侧边栏展开/收起动画（200ms ease-out）
- 卡片 hover 时微上浮（translateY(-2px)）+ 阴影加深
- 按钮点击保留 press-pop 反馈（scale 0.93 + inset shadow）
- 所有动画包含 prefers-reduced-motion 媒体查询兜底
- 列表项进入时保留 stagger 渐入动画

## Non-Functional Requirements
- **NFR-1 (Performance)**: 首屏加载不阻塞，路由级代码分割保留；动画保持 60fps
- **NFR-2 (Accessibility)**: 所有交互元素有 aria-label；焦点状态不被移除；颜色对比度满足 WCAG AA 标准（参考 Twitter/X）
- **NFR-3 (Responsive)**: 本任务仅针对桌面端（≥1024px），移动端布局不改动
- **NFR-4 (Consistency)**: 所有颜色/字体/间距使用现有设计系统 token，不引入新的硬编码颜色值
- **NFR-5 (Build)**: 重构后 `npm run build` 必须成功，无 TypeScript 错误

## Constraints
- **Technical**: React 18 + TypeScript + Tailwind CSS v4 + Vite + React Router v6 + Lucide React 图标；不引入新依赖
- **Business**: 保持现有路由路径不变，避免破坏用户书签和分享链接
- **Dependencies**: 复用现有 mockData 服务和 api 服务，不新增 mock 数据字段（如有需要可在组件内扩展 mock）

## Assumptions
- Hero Banner 的 3D 插画区域可先用渐变背景 + 简单图标占位，后续替换正式素材
- 数据统计卡片的数字先用 mock 数据（128条/12个话题），后续接入真实 API
- 推荐用户数据先用 mock 数据
- 左侧导航的"我的收藏/浏览历史/草稿箱"点击后暂时跳转到占位页面或 profile 页（因为这些页面当前不存在完整功能）
- 通知铃铛点击后暂时无弹窗（仅 UI 展示），后续再做通知中心

## Acceptance Criteria

### AC-1: 全局顶栏可见且功能完整
- **Given**: 用户在任意桌面端页面
- **When**: 页面加载完成
- **Then**: 顶部显示固定导航栏，包含 Logo、居中文搜索框、发布按钮、通知图标、用户头像
- **Verification**: `human-judgment`
- **Notes**: 顶栏高度约 60px，搜索框宽度约 480px 居中

### AC-2: 左侧导航内容与设计一致
- **Given**: 用户在桌面端
- **When**: 左侧栏展开时
- **Then**: 可见菜单项顺序为：推荐(⭐)、关注、求证、辩论场、吃瓜榜、公益、话题、(分隔线)、我的收藏、浏览历史、草稿箱；底部有数据统计卡片和版权信息
- **Verification**: `human-judgment`

### AC-3: 左侧栏收起/展开交互正常
- **Given**: 左侧栏当前为展开状态
- **When**: 点击收起按钮
- **Then**: 侧栏平滑收缩为 64px 宽图标栏，菜单项只显示图标；点击展开按钮恢复
- **Verification**: `programmatic` + `human-judgment`

### AC-4: 右侧面板四模块完整
- **Given**: 用户在首页
- **When**: 查看右侧面板
- **Then**: 从上到下依次显示：正在激辩（前3红编号）、吃瓜榜（第1名带热标签）、热门话题（# 前缀）、推荐用户（带关注按钮）；每个模块有"更多>"入口
- **Verification**: `human-judgment`

### AC-5: 首页双列瀑布流布局正确
- **Given**: 用户访问首页
- **When**: 页面加载
- **Then**: 内容区为等宽双列，卡片按左右交替排列；图片高度自然变化不统一
- **Verification**: `human-judgment`

### AC-6: 卡片状态标签正确显示
- **Given**: 有不同状态的内容卡片
- **When**: 卡片渲染
- **Then**: 图片左上角显示对应状态标签：热门(红)、已求证(绿)、求证中(黄)、辩论中(紫)；标签为圆角小角标样式
- **Verification**: `human-judgment`

### AC-7: 一级/二级分类 Tab 切换正常
- **Given**: 用户在首页
- **When**: 点击"热门"一级 Tab
- **Then**: 切换到热门内容流，激活项有红色下划线指示；二级分类可点击切换
- **Verification**: `human-judgment`

### AC-8: 构建无错误
- **Given**: 代码重构完成
- **When**: 运行 `npm run build`
- **Then**: 构建成功，无 TypeScript 类型错误，无 import 错误
- **Verification**: `programmatic`

### AC-9: 其他页面适配新全局布局
- **Given**: 用户访问求证页/辩论场/个人中心等
- **When**: 页面加载
- **Then**: 仍使用新的全局顶栏+侧栏+右面板布局，页面内容在新框架下正常显示，不出现布局错乱
- **Verification**: `human-judgment`

### AC-10: 无全局样式污染
- **Given**: 新样式已应用
- **When**: 检查 Tailwind 工具类是否被全局 CSS 覆盖
- **Then**: 全局 CSS 不覆盖 Tailwind 响应式/工具类；毛玻璃效果 blur ≥20px；焦点可见样式正常
- **Verification**: `programmatic` + `human-judgment`
